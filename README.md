# Arduino UNO — два проекта на голой плате

Два независимых скетча для двух Arduino UNO. Никаких внешних
компонентов, только плата + USB. Всё взаимодействие — через Serial.

| Папка | Что это | Терминал |
| ----- | ------- | -------- |
| [`uno_os/`](uno_os/)                         | **UNO OS v0.1** — однопользовательская «ОС»: shell с 22 командами, файловая система в EEPROM (1 КБ, 8 файлов), линейный редактор, встроенный **mini-Python интерпретатор** (переменные, арифметика, `if`/`while` с отступами, `print`, GPIO), autoexec файла `boot`. В подпапке [`host/`](uno_os/host/) лежит `unoctl.py` — Python-клиент-терминал с автодетектом порта (по VID/PID) и загрузкой/скачиванием файлов через меню `Ctrl+]`. | `python unoctl.py` (или любой Serial Monitor, 115200 бод) |
| [`50_ascii_raycaster/`](50_ascii_raycaster/) | Псевдо-3D «дум-подобный» raycaster в ASCII 40×18, рисуется в терминал. WASD-управление. | PuTTY / Windows Terminal, 115200 бод |

## Заливка

Каждая папка — отдельный скетч Arduino IDE. Имя `.ino` совпадает с
именем папки — это требование IDE.

1. Открой нужный `.ino` в Arduino IDE.
2. Tools → Board → Arduino UNO.
3. Tools → Port → COM нужной платы.
4. Upload.

Скетчи можно (и нужно) залить на разные платы — они никак не
взаимодействуют между собой.

## Что в каждой папке

- `*.ino` — скетч.
- `README.md` — инструкция, объяснение, примеры использования.

## Что куда подходит

**UNO OS** — гик-проект «настоящей» ОС в 32 КБ flash и 2 КБ RAM:
файловая система в EEPROM (1 КБ), shell с командами, линейный редактор,
встроенный **mini-Python интерпретатор** для запуска программ из файлов
(переменные `a..z`, арифметика, `if`/`while` с отступами, GPIO). Файлы
переживают перезагрузку. Можно настроить autoexec через файл `boot`.
Ощущается как мини-DOS из 80-х, только с Python-синтаксисом. К нему идёт
**Python-клиент `unoctl.py`** — сам находит подключённую плату по
VID/PID, открывает порт, и параллельно с обычным терминалом умеет
заливать/скачивать файлы через встроенное меню (`Ctrl+]`).

**ASCII Raycaster** — 3D-рендер лабиринта в текстовом терминале.
Технологически интересен (DDA-обход сетки, fisheye-коррекция), но
требует терминала с поддержкой ANSI escape-кодов — Arduino IDE Serial
Monitor не подойдёт.

## Поддерживаемые платы

Тестировалось на **Arduino UNO R3** (ATmega328P, 16 МГц, 32 КБ flash,
2 КБ RAM, 1 КБ EEPROM). Должно работать и на UNO R4, Nano, Pro Mini,
но размеры буферов настроены под 2 КБ RAM. На платах побольше (Mega,
ESP32) можно поднять лимиты — см. `#define` в начале каждого скетча.
