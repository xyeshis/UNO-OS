# UNO OS v0.1

Однопользовательская «операционная система» для голой Arduino UNO.
32 КБ flash, 2 КБ RAM, 1 КБ EEPROM. Никакой обвязки — только плата + USB.
Всё взаимодействие — через Serial.

```
UNO OS v0.1  -  type 'help'
ram free: 1462 b   files: 0/8
uno> help
help  ls  cat  write  rm  mv  cp  df  mem  echo
run  py  format  clear  reboot  ver  uptime
pinm  pinw  pinr  aread  led
uno> write hi.py
end with '.'  (':q' to abort)
... > x = 0
... > while x < 5:
... >   print(x)
... >   x = x + 1
... > .
wrote 38 b
uno> py hi.py
0
1
2
3
4
uno>
```

## Что внутри

- **shell** с 22 командами; редактируемая строка (backspace, Ctrl+C);
- **файловая система в EEPROM**: 8 файлов × до 116 байт данных, имена ≤ 8;
- **редактор** `write` (line editor: точка `.` сохраняет, `:q` отменяет);
- **встроенный mini-Python интерпретатор**:
  - переменные `a..z` (int16),
  - арифметика и сравнения, `&&` `||` `!`,
  - блоки `if`/`while` по отступам (как в настоящем Python),
  - `print(...)`, `delay(ms)`, `pinm/pinw/pinr`, `aread`, `led(0/1)`, `ms()`,
  - комментарии `# ...`;
- **autoexec** файла `boot` при старте (свой rc.local);
- **программный reboot** через WDT.

К плате идёт **Python-клиент `host/unoctl.py`** — он же терминал, в котором
всё происходит: команды набираешь там же, вывод UNO там же. Дополнительно
по `Ctrl+]` открывается меню для загрузки/скачивания файлов в EEPROM-FS.

## Структура

```
uno_os/
├── uno_os.ino          ← скетч, заливается в Arduino IDE
├── README.md           ← этот файл
└── host/
    ├── unoctl.py       ← Python-клиент-терминал (CLI)
    ├── unoctl_gui.py   ← Tkinter GUI поверх него
    └── requirements.txt
```

## Заливка скетча

1. Открой `uno_os.ino` в Arduino IDE.
2. Tools → Board → **Arduino UNO**.
3. Tools → Port → твой COM-порт.
4. Жми **Upload**.

Размер: ~14 КБ flash, ~280 байт SRAM статически. Запас комфортный.

## Подключение через `unoctl.py`

В этой папке — самодостаточный Python-клиент. Сам ищет UNO по VID/PID,
открывает Serial @ 115200 и работает как живой терминал.

```sh
cd uno_os/host
pip install -r requirements.txt
python unoctl.py
```

Получаешь:

```
[i] COM5 @ 115200
[i] терминал. Ctrl+] — меню (upload/download/quit).
UNO OS v0.1  -  type 'help'
ram free: 1462 b   files: 0/8
uno>
```

### GUI-режим — `unoctl_gui.py`  (Windows XP edition)

Если хочется потыкать мышкой — рядом лежит Tkinter-окно, **полностью
стилизованное под Windows XP Luna**. Своя оконная рамка, синий градиентный
title bar с кнопками `[_][□][✕]`, бежевый client area, raised XP-кнопки в
группах (`Connection`, `Shell`, `Files & scripts`, `GPIO`, `System`),
терминал в стиле классического `cmd.exe` (чёрный фон, Lucida Console,
silver/gray текст), status bar и **декоративный XP-таскбар** внизу с
зелёной кнопкой **start** (с настоящим start-меню), плиткой `UNO OS — GUI`
и часами.

```sh
python unoctl_gui.py
```

Tkinter входит в стандартную поставку Python — доустанавливать ничего не
надо.

Что внутри:

- **Своё окно** — title bar с XP-градиентом, перетаскивание мышкой, double-
  click по title bar = max/restore. Кнопка `_` сворачивает (использует
  iconify, временно снимая override), `□` максимизирует, `✕` закрывает.
- **Menu bar** в стиле XP: `File · Edit · View · Tools · Help` —
  оранжевый hover, разделители, вложенные пункты.
- **Toolbar** с XP-style group boxes:
  - `Connection`: COM-порт + `↻` rescan + `Connect/Disconnect`,
  - `Shell`: `ls`, `df`, `mem`, `ver`, `uptime` — одной кнопкой,
  - `Files & scripts`: `Upload…`, `Download…`, `Run…`, `Edit…`,

- **Терминал** в чёрной cmd.exe-теме, с цветными тегами для info / err.
- **Поле ввода** с XP-style prompt `C:\UNO>`, стрелки `↑/↓` — история,
  `Enter` — отправить, `Ctrl+C` шлёт `\x03` на плату (прервать скрипт).

- **Фейковая XP-таскбар** в самом низу: зелёная кнопка `start` (открывает

About-окно (Help → About UNO OS) тоже в XP-стиле — с `OK` кнопкой и

### CLI-режим (без интерактива)

```sh
python unoctl.py --list-ports                # показать все COM-порты
python unoctl.py --port COM5                 # форсировать порт
python unoctl.py --upload   blink.py boot    # залить локальный → в FS UNO
python unoctl.py --download boot dump.py     # скачать из FS UNO → локально
```

### Меню `Ctrl+]`

Прямо из терминала, не отключаясь от UNO:

```
[ uno menu | u local [remote] | d remote [local] | l | x ]
> u blink.py boot
[i] upload: 38 b  ->  boot
[ -> terminal ]
uno>
```

Команды меню: `u` upload, `d` download, `l` показать локальные файлы,
`x` выход, пустая строка — обратно в терминал.

### Любой другой Serial-терминал

UNO OS — обычный текстовый протокол на 115200 бод, без эха со стороны
терминала. Заработает в Arduino IDE Serial Monitor, PuTTY, Windows
Terminal через любой serial-плагин, `tio`, `screen`, `minicom`. Просто
для удобной заливки/выгрузки файлов нужен `unoctl.py`.

## Команды shell

| Команда           | Что делает                                        |
| ----------------- | ------------------------------------------------- |
| `help`            | список команд                                     |
| `ls`              | файлы и их размер                                 |
| `cat <name>`      | напечатать содержимое файла                       |
| `write <name>`    | записать/перезаписать файл (line editor)          |
| `rm <name>`       | удалить файл                                      |
| `mv <old> <new>`  | переименовать                                     |
| `cp <src> <dst>`  | копия файла                                       |
| `df`              | сколько занято в EEPROM                           |
| `mem`             | сколько свободно RAM                              |
| `echo <text>`     | напечатать текст                                  |
| `run <name>`      | запустить mini-Python скрипт (`py` — алиас)       |
| `py <name>`       | то же самое                                       |
| `format`          | стереть всю FS (с подтверждением)                 |
| `clear` / `cls`   | очистить экран (ANSI escape)                      |
| `reboot`          | программный reset через watchdog                  |
| `ver`             | версия и параметры                                |
| `uptime`          | время с включения                                 |
| `pinm <p> <m>`    | `pinMode`: 0=INPUT, 1=OUTPUT, 2=INPUT_PULLUP      |
| `pinw <p> <0/1>`  | `digitalWrite`                                    |
| `pinr <p>`        | `digitalRead` → печать                            |
| `aread <p>`       | `analogRead` → печать                             |
| `led on/off`      | встроенный LED (D13)                              |
| `mine` / `mines` / `ms` | встроенная игра «Сапёр» в терминале         |

### Сапёр в терминале

Встроенная мини-игра прямо в shell, без GUI. Поле 8×8, 10 мин.

```
uno> mine
```

Управление:

- `W A S D` — движение курсора
- `O` или `Space` — открыть клетку
- `F` — поставить/снять флаг
- `R` — новая игра
- `Q` — выход обратно в shell

### Line editor

`write filename` запускает простой построчный редактор. Каждая введённая
строка пишется в файл вместе с `\n`. Точка `.` на отдельной строке —
сохранить и выйти. `:q` — отмена (файл не создаётся / не меняется).

```
uno> write hi
end with '.'  (':q' to abort)
... > hello uno
... > .
wrote 10 b
```

## mini-Python

Каждая строка — один statement. Блоки — по отступам, как в нормальном
Python. Любое количество пробелов ≥ 1 (рекомендуется 2). Табы тоже ок,
но не смешивай.

### Поддержано

| Конструкция                                      | Пример                       |
| ------------------------------------------------ | ---------------------------- |
| Переменные `a..z` (int16, −32768..32767)         | `x = 5`                      |
| Арифметика                                       | `x + y * 2`                  |
| Деление и остаток                                | `n / 3`, `n % 7`             |
| Сравнения                                        | `x == 5`, `n != 0`, `x <= y` |
| Логика                                           | `a && b`, `a || b`, `!x`     |
| `print(expr)` / `print("text")` / `print()`      | `print(x + 1)`               |
| `delay(ms)`                                      | `delay(500)`                 |
| `pinm(pin, mode)` (0/1/2)                        | `pinm(13, 1)`                |
| `pinw(pin, val)`                                 | `pinw(13, 1)`                |
| `pinr(pin)`, `aread(pin)`, `ms()`                | в выражениях                 |
| `led(0/1)`                                       | `led(1)`                     |
| `if cond:` блок                                  | см. ниже                     |
| `while cond:` блок                               | см. ниже                     |
| Комментарии `# ...`                              | `# blink LED`                |

### Не поддержано (намеренно)

- `def`, `return` — нет функций пользователя;
- строки как значения (только литерал внутри `print("...")`);
- списки, словари, кортежи;
- `for ... in ...` (используй `while`);
- `else:` (его реально жалко, но это сожрало бы flash);
- многобуквенные имена переменных.

Это не Python — это **Python-flavored DSL**, чтобы синтаксис был знакомым.

### Пример: счётчик

```python
i = 0
while i < 10:
  print(i)
  i = i + 1
print(-1)
```

### Пример: blink через встроенный LED

```python
i = 0
while i < 20:
  led(1)
  delay(150)
  led(0)
  delay(150)
  i = i + 1
```

### Пример: реакция на кнопку (D2 → INPUT_PULLUP, кнопка на GND)

```python
pinm(2, 2)
pinm(13, 1)
while 1:
  if pinr(2) == 0:
    pinw(13, 1)
  if pinr(2) != 0:
    pinw(13, 0)
```

### Пример: чтение АЦП с гистерезисом

```python
pinm(13, 1)
while 1:
  v = aread(0)
  if v > 700:
    led(1)
  if v < 300:
    led(0)
  delay(50)
```

## Autoexec — файл `boot`

Если в FS есть файл с именем ровно `boot`, его содержимое автоматически
запускается через mini-Python сразу после старта. Это аналог `rc.local`
или `autoexec.bat`.

```
uno> write boot
end with '.'  (':q' to abort)
... > led(1)
... > delay(300)
... > led(0)
... > .
wrote 26 b
uno> reboot
```

После загрузки UNO мигнёт встроенным LED и вернётся к prompt’у. Если
boot-скрипт зацикленный (`while 1:`), он будет крутиться вечно и shell
не вернётся — это фича. Чтобы выйти из этого состояния, можно прошить
скетч заново или через `unoctl.py` залить новый `boot` (он перезапишет
старый прямо во время работы — но скрипт уже исполняется, поэтому
помогает только `reboot`-команда после).

## Внутренности

### EEPROM FS

8 слотов по 128 байт, всего ровно 1024 = 1 КБ EEPROM.

| Байты   | Значение                                     |
| ------- | -------------------------------------------- |
| 0       | magic `0xA5` (used) или `0xFF` (free)        |
| 1..8    | имя (8 b, NUL-padded)                        |
| 9..10   | размер payload (uint16 LE)                   |
| 11      | reserved                                     |
| 12..127 | данные (116 b)                               |

Свежий чип уже «отформатирован» — все байты `0xFF`, что и означает
«слот свободен». Поэтому стартовая инициализация не нужна.

`EEPROM.update()` пишет только при реальном изменении байта — экономит
ресурс ячеек (Atmel заявляет 100 000 циклов).

### mini-Python — как работает

Скрипт читается в RAM-буфер (≤ 120 b) при `run`. Парсер — рекурсивный
спуск с приоритетами (mul → add → cmp → and → or). Блоки определяются
отступами: при входе в `while` запоминаем позицию заголовка строки и
индент; когда внешний цикл видит, что отступ следующей строки ≤ инденту
заголовка, он прыгает обратно к заголовку и заново вычисляет условие.
Стек блоков — до 6 уровней (`PY_STACK = 6`). Никакого AST или байткода
нет — прямая интерпретация прямо по тексту.

### Reboot через WDT

Команда `reboot` включает watchdog с таймаутом 15 мс и зависает в
бесконечном цикле — через 15 мс чип перезагружается. В `setup()` мы
сразу делаем `MCUSR=0; wdt_disable();`, чтобы не было ресет-петли.

## Ограничения

- Файл — до **116 байт**. Это намеренно: вся FS в 1 КБ EEPROM.
- Имена файлов — до **8 символов**, `[a-zA-Z0-9._-]`.
- Переменные mini-Python — **a..z**, тип **int16**.
- Длина строки в shell и редакторе — **80 байт**.
- Глубина блоков mini-Python — **6** (`if`/`while` суммарно).
- Один скрипт исполняется блокирующе. Asyncio тут нет.

## Идеи на доработку

- `for i in range(n):` как сахар над `while`.
- Несколько 16-bit массивов фиксированного размера для «списков».
- Многобуквенные идентификаторы (потребует таблицу имён → индексов).
- Аппаратные UART/SPI/I2C обёртки в виде `pinr`/`aread`-style функций.
- Кооперативные задачи: фоновый скрипт + интерактивный shell одновременно.
- Счётчик циклов записи EEPROM в заголовке слота — wear leveling по факту.
